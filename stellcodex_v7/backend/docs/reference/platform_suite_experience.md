# Platform Suite Experience

STELLCODEX should feel like one calm, simple entry point even though it
contains many applications.

Core rule:

- STELLCODEX is the product
- applications are focused work surfaces inside the product

Suite interaction rules:

- the first step should stay obvious: upload a file, open a project, or enter a
  focused application
- file uploads should route directly into the responsible application
- the active application should get the widest stage when the left rail
  collapses
- 3D review, 2D drawing review, and document review must not share the same
  control density or wording
- files, projects, and share controls should remain suite-level services
- do not create copy pages that repeat the same launch flow
- do not place the same primary button twice on one surface
- keep copy English-first and readable without repo context

Packaging rules:

- Free, Plus, and Pro describe suite access levels
- separate mobile or store builds may expose one focused application
- separate builds must still map to the same STELLCODEX identity and shared
  platform core
