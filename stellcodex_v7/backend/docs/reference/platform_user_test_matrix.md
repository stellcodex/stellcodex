# Platform User Test Matrix

This reference defines the user-scenario matrix for the STELLCODEX suite
experience.

The matrix is intentionally broad:

- 10 different personas
- 10 different suite tasks
- 100 scenario combinations

The purpose is to prove that the platform remains understandable for:

- low-confidence first-time users
- rushed users
- technical power users
- visually oriented reviewers
- detail-heavy reviewers
- skeptical users who need trust signals before they proceed

## Persona groups

1. cautious novice
2. rushed manager
3. visual inspector
4. detail-focused drafter
5. skeptical founder
6. shop-floor operator
7. procurement lead
8. quality reviewer
9. power analyst
10. multilingual beginner

## Task groups

1. upload a 3D model
2. upload a 2D drawing
3. upload a document
4. open the files and sharing hub
5. open projects
6. discover the applications catalog
7. run conversion
8. run mesh generation
9. configure MoldCodes
10. open the library

## Contract expectations

- every upload scenario must route into the responsible application
- every scenario must map to one canonical surface
- the suite home must stay simple and trust-building
- the suite must remain English-first
- Free, Plus, and Pro must remain the only public plan tiers
- copy pages and duplicate primary-entry flows are forbidden
