# Suite Visual Language

STELLCODEX should stay visually simple even while the product expands into many
separate applications.

## Locked baseline

- Background: white or near-white surfaces only
- Primary text: near-black for high readability
- Accent: teal (`#0f766e`) for one clear call-to-action system
- Sidebar behavior: collapsible so the active application can take the widest area
- Primary action rule: one obvious main action per screen

## What this protects

- Users should not have to decode color meaning across apps.
- The suite should feel reliable to first-time users with low technical confidence.
- 3D, 2D, documents, files, and sharing can differ by workflow without looking like separate products.

## Forbidden drift

- Dark auth portals disconnected from the main suite
- Duplicate primary buttons on one screen
- Mixed product names such as `STELLCONSOLE`
- Copy pages that repeat the same entry flow with different wording
